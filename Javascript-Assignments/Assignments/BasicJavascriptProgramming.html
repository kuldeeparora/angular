  • Operators
    • Additive(+,-) The term additive operators includes both addition(+) and subtraction(-) operators
    • Multiplicative(*, /) they perform multiplication(*) and division(/)
    • Equality(==, ===, !=, !==) You use equality operators to test whether two expressions are the same or different.
    • Relational(>, <, >=, <=, in, instanceof) Relational operators test expressions to find out whether they are greater than or less than each other, or whether a given value is in a list or is an instance of a certain type.

  if (myDate instanceof Date) {

    • Unary(delete, void, typeof, ++, --, +, -, ~, !) Unary operators have a single operand or work with a single expression in JavaScript.
      ++ and -- operators to increment and decrement a number 
      var aNum = 4;
var y = aNum++;  // y now has the value 4, but aNum then has the value 5
The second example is prefixing:
var aNum = 4;
var y = ++aNum;  // y now has the value 5, as does aNum
      Converting to a number with the plus sign 
      var x = +"43";
This code results in the string “43” being converted to a number by JavaScript 
      and the numeric value 43 being stored in the variable x.
      Using the delete operator 
      var myArray = ["The RCMP", "The Police", "State Patrol"];
delete myArray[0];  // myArray now contains only "The Police" and "State Patrol"
The preceding code creates an array called myArray and then promptly deletes the 
      value at the first index, making it “undefined.”

    • Bitwise Operators(&, |, ^, ~, <<, >>, >>>) Bitwise operators include AND(&), OR(|), XOR(^), NOT(~), Shift Left(<<), Shift Right With Sign(>>), and Shift Right With Zero Fill(>>>)
      negating with bitwise not and logical not 
      Bitwise AND	a & b	Returns a one in each bit position for which the 
      corresponding bits of both operands are ones.
Bitwise OR	a | b	Returns a one in each bit position for which the 
      corresponding bits of either or both operands are ones.
Bitwise XOR	a ^ b	Returns a one in each bit position for which the 
      corresponding bits of either but not both operands are ones.
Bitwise NOT	~ a	Inverts the bits of its operand.
Left shift	a << b	Shifts a in binary representation b (< 32) bits to 
      the left, shifting in zeros from the right.
Sign-propagating right shift	a >> b	Shifts a in binary representation 
      b (< 32) bits to the right, discarding bits shifted off.
Zero-fill right shift	a >>> b	Shifts a in binary representation b (< 32) bits
       to the right, discarding bits shifted off, and shifting in zeros from the left.
